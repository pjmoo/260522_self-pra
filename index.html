01

🔄 반복문 — for / while

같은 코드를 여러 번 실행할 때 사용. 조건이 true인 동안 계속 반복한다.

🔁

while 반복문

조건이 true인 동안 계속 실행. 반드시 변수를 변화시켜서 언젠가 조건이 false가 되게 해야 무한루프를 막을 수 있다.

➡️

for 반복문

초기식, 조건식, 증감식을 한 줄에 작성. 반복 횟수가 명확할 때 주로 사용한다.

⏭️

break / continue

break는 반복문 전체 탈출, continue는 현재 회차만 건너뛰고 다음 반복을 계속한다.

초기식 (let i = 0)
→
조건식 (i < 5)
→
실행 블록 { }
→
증감식 (i++)
🔄
조건 false → 종료
01_for.js 
// while — 조건이 만족될 때까지 반복
let w = 0;
while (w < 5) {
  console.log(w);
  w++;  // 이게 없으면 무한반복 ⚠️
}

// for (초기식 ; 조건식 ; 증감식) {}
for (let i = 0; i < 5; i++) {
  console.log(i);  // 0, 1, 2, 3, 4
}

// 배열 순회 — break / continue 활용
const arr = ["아침", "간식", "점심", "간식", "저녁"];
for (let i = 0; i < arr.length; i++) {
  if (arr[i] === "간식") continue; // 간식은 건너뜀
  console.log(arr[i]);
  if (arr[i] === "점심") break;   // 점심 이후 중단
}
// 출력: 아침 → 점심 (간식 skip, 저녁 안 나옴)

// 2차원 배열 — 중첩 for (행 × 열)
const matrix = [
  [1, "윌리엄", "바나나를 좋아함"],
  [2, "김자바", "김을 좋아함"],
];
for (let i = 0; i < matrix.length; i++) {
  for (let j = 0; j < matrix[i].length; j++) {
    console.log(matrix[i][j]);
  }
}
⚠️
배열 인덱스는 0부터!  마지막 인덱스 = 배열 길이 - 1. 그래서 조건식은 i < arr.length (미만), i <= arr.length - 1 (이하) 둘 다 같다.
02

🔑 for...in / for...of

for 반복문의 특별한 형태. 객체와 배열을 더 편리하게 순회한다.

구분 for...in for...of
가져오는 것 키 (key) — 인덱스, 프로퍼티명 값 (value) — 실제 데이터
주로 쓰는 대상 객체 { } 의 키를 탐색 배열, 문자열 등 iterable
일반 객체에 사용 ✅ 가능 ❌ TypeError — Object.values() 필요
변수 선언 const key (매 반복마다 새로 초기화) const v (매 반복마다 새로 초기화)
02_for+in+of.js 
const arr = ["연어", "아보카도", "스테이크"];
const obj = { name: "윌리엄", job: "반찬가게 사장", salary: 300_000 };

// for...in → 키를 순회
for (const key in arr) {
  console.log(key, arr[key]); // 0 연어 / 1 아보카도 / 2 스테이크
}
for (const key in obj) {
  console.log(key, obj[key]); // name 윌리엄 / job ... / salary ...
}

// for...of → 값을 순회 (배열에서 주로 사용)
for (const v of arr) {
  console.log(v); // 연어 / 아보카도 / 스테이크
}

// 객체는 for...of 바로 못 씀 → Object.values() 활용
for (const v of Object.values(obj)) {
  console.log(v); // 윌리엄 / 반찬가게 사장 / 300000
}
💡
배열에는 보통 for...of가 더 안전하다. for...in은 배열에 임의로 추가된 숫자가 아닌 키도 순회해서 예상치 못한 결과가 나올 수 있다.
03

⚙️ 함수 기본

코드를 재사용 가능한 단위로 묶은 것. function 키워드로 선언하고, 이름으로 호출한다.

📥

return — 반환값

함수는 실행 결과를 return으로 돌려준다. return을 만나면 즉시 함수가 종료된다. (함수 내부의 break 역할)

🔐

지역 변수

함수 안에서 선언된 변수 (let b)는 함수 밖에서 접근 불가. 함수가 끝나면 사라진다.

🎲

다양한 반환 타입

하나의 함수가 조건에 따라 배열, 객체, 문자열 등 다른 타입의 값을 반환할 수 있다.

03_function.js 
let a = 0;

function myFun() {
  console.log(a);    // 바깥 변수 a 접근 가능
  a++;
  let b = "나는 함수 안에서 선언된 변수";
  return b;          // b를 반환하고 함수 종료
}

myFun();             // 실행만
const c = myFun(); // 반환값을 변수에 저장
console.log(c);     // "나는 함수 안에서 선언된 변수"
// console.log(b) → ❌ ReferenceError: b is not defined

// switch + return: if/else 없이 분기 처리
function myFun3() {
  const r = Math.random();
  switch (true) {
    case r > 0.9: return [1, 2, 5];        // 배열
    case r > 0.5: return { a: 10 };        // 객체
    default:      return "동전던지기 실패"; // 문자열
  }
}
04

📨 매개변수 (Parameter)

함수 밖 → 안으로 값을 전달하는 통로. 매개변수(parameter)는 받는 쪽, 인자(argument)는 넘기는 값.

기능문법설명
기본값 (Default)function f(a = 120)인자 안 넘기면 기본값 사용
argumentsarguments 객체함수에 들어온 모든 인자를 담은 유사배열
나머지 (Rest)function f(a, ...b)나머지 인자들을 배열로 받음. 마지막 위치에만 가능
객체 매개변수function f(props = {})여러 옵션을 객체 하나로 묶어 전달 (DTO 패턴)
04_parameter.js 
// JS 함수는 인자 수가 달라도 에러 안남
function f1(a, b, c) { console.log(a*2, b*2, c*2); }
f1();         // NaN NaN NaN (undefined*2)
f1(1, 2);    // 2 4 NaN
f1(1,2,3,4); // 4번째 인자는 무시

// 기본값 (Default Parameter)
function f2(a = 120, b = 150) { return a * b; }
console.log(f2());     // 18000
console.log(f2(100)); // 15000 (a만 덮어씀)

// rest 연산자 — 나머지를 배열로
function f4(a, ...b) {
  console.log("a", a); // 첫번째 인자
  console.log("b", b); // 나머지 인자들의 배열
}
f4(1, 2, 3, 4); // a: 1, b: [2, 3, 4]

// 객체로 받기 (옵션 패턴)
function f5(props = {}) {
  if (props.a) console.log("props.a", props.a);
  console.log("props.b", props.b || "없음");
}
f5({ a: 10 }); // props.a 10 / props.b 없음
ℹ️
나머지 연산자(...b)는 반드시 마지막 매개변수여야 한다. console.log(...data)가 바로 이 스프레드/레스트 문법을 활용한 예시다.
05

🏹 화살표 함수 / 함수 표현식 / 익명함수

함수를 선언하는 다양한 방법. 각각 동작 방식과 this 바인딩이 다르다.

종류문법특징
함수 선언문function a() {}호이스팅 O — 선언 전에 호출 가능
함수 표현식const f = function() {}호이스팅 X — 선언 후에만 호출 가능
익명함수 즉시실행(function() {})()이름 없이 바로 실행 (IIFE)
화살표 함수const f = () => 값짧은 문법, 자신만의 this 없음
05_function2.js 
// 익명함수 즉시실행 (IIFE)
(function () {
  console.log("나는 존재한다");
})();

// 함수 표현식 — 함수를 값처럼 변수에 저장
const f = function (fn) {
  fn();                     // 매개변수로 받은 함수 실행
  console.log("나는 존재한다 2");
};
f(function () {
  console.log("포케 먹기 싫다"); // 함수를 값으로 전달
});

// 화살표 함수 — (입력) => 출력
const af  = () => 0;                        // 값만 반환 (return 생략)
const af2 = (a) => console.log(`실행: ${a}`); // 한 줄 실행
const af3 = (a) => {             // 블록 사용 시 return 명시
  a++;
  a *= 2;
  return a;
};
console.log(af3(100)); // 202
⚠️
함수 선언문은 호이스팅된다! 코드 어디서든 호출 가능. 하지만 const/let + 화살표함수/표현식을 쓰면 호이스팅 걱정 없이 선언 순서를 명확히 할 수 있다.
06

📦 구조 분해 할당 (Destructuring)

객체나 배열의 값을 한 번에 여러 변수로 뽑아내는 문법. 코드가 훨씬 짧아진다.

🗂️

객체 구조 분해

키 이름으로 값을 뽑는다. 별칭(name: newName), 기본값(pet = "고양이"), 나머지(...rest) 모두 가능.

📋

배열 구조 분해

순서대로 값을 뽑는다. 중간을 ,로 건너뛸 수 있고, 나머지는 ...rest로 받는다.

🔄

얕은 복사 vs 재할당

obj2 = obj는 같은 주소(재할당). {...obj}는 새 객체(얕은 복사). 중첩 객체는 여전히 참조 공유!

06_da.js 
// 객체 구조 분해
const obj = { name: "윌리엄", age: 18, job: "반찬가게 직원" };
const { name, age, job } = obj;  // 키 이름으로 꺼내기
const { name: personName } = obj;  // 별칭 지정
const { salary, pet = "고양이", ...rest } = obj;
// salary: undefined (없는 키), pet: "고양이" (기본값), rest: 나머지

// 배열 구조 분해
const arr = ["삼성전자", "SK하이닉스", "SK스퀘어", "현대차"];
const [a1, a2, a3, a4] = arr;
const [, , , , , a7] = arr; // 앞을 ,로 생략

// 값 교환 트릭!
let i = 0, j = 100;
[j, i] = [i, j]; // i=100, j=0 (임시변수 없이 스왑!)

// 얕은 복사
const obj2 = obj;          // 재할당 — 같은 객체 가리킴
const obj3 = { ...obj };  // 얕은 복사 — 새 객체 생성
// 단, 내부에 중첩 객체가 있으면 그 참조는 공유됨 (깊은복사 ≠ 얕은복사)
07

⬆️ 호이스팅 (Hoisting)

선언부가 코드 실행 전에 메모리에 먼저 등록되는 현상. "코드를 올린다(hoist)"는 의미.

코드 파싱
→
var / function 선언 수집
→
메모리에 등록
→
코드 순서대로 실행
구분호이스팅?초기값선언 전 접근
var✅undefined에러 없이 undefined
let / const⛔ (TDZ)없음ReferenceError
함수 선언문✅ (완전)함수 자체호출 가능
함수 표현식 / 화살표⛔없음ReferenceError
07_hoisting.js 
// 함수 선언 전에도 호출 가능! (호이스팅)
console.log(typeof fun); // "function"
console.log(fun());       // "덮어 씌웠지롱" (나중 선언이 덮음)
console.log(bar);        // undefined (var는 값 없이 먼저 등록됨)

var bar = 200;

function fun() { return 0; }
function fun() { return "덮어 씌웠지롱"; } // 같은 이름 재선언 가능!

// ✅ 해결책: const + 화살표함수 사용
// const fun = () => 0; // 재선언도, 호이스팅도 방지
💡
실무 팁: const/let과 화살표함수/함수표현식만 써도 호이스팅 문제를 만날 일이 거의 없다.
08

🔭 스코프 (Scope)

변수가 어디서 접근 가능한지를 결정하는 범위. "어디서 보이냐"의 문제.

전역 스코프 (Global)

어디서든 접근 가능한 변수들 (var v1, let v3)

함수 스코프 (Function)

var는 함수 블록만 격리. 함수 밖으로 못 나감

블록 스코프 (Block)

let / const는 { } 블록 내에만 존재. 반복문, 조건문, 함수 모두 해당

08_scope.js 
// var: 함수 스코프 — 블록 { } 밖으로 나옴
{
  var v1 = "나는 var야";   // 블록 밖에서도 접근 가능
  let v2 = "나는 let이야"; // 블록 밖에서 접근 불가
}
console.log(v1); // "나는 var야" ✅
// console.log(v2); ❌ ReferenceError: v2 is not defined

// for 루프의 i는 블록 스코프
for (let i = 0; i < 5; i++) { console.log(i); }
// console.log(i); ❌ ReferenceError: i is not defined

// 함수 스코프
var vv2 = "바깥";
function f1() {
  var vv1 = "나는 함수 안"; // 함수 밖 접근 불가
  vv2 = "바깥 변수를 바꿨어"; // 바깥 변수 수정은 가능
}
f1();
// console.log(vv1); ❌  /  console.log(vv2); ✅ 바깥 변수를 바꿨어
ℹ️
블록은 바깥에 결과물을 직접 전달 못한다. 하지만 이미 바깥에 선언된 변수에 값을 쓰거나, return으로 함수에서 값을 내보낼 수 있다.
09

🔒 클로저 (Closure)

함수가 선언된 시점의 스코프를 기억하는 것. 함수가 종료된 뒤에도 바깥 변수를 살아있게 한다.

📍

렉시컬 스코프

함수는 호출된 위치가 아니라 선언된 위치의 스코프를 기준으로 동작한다. (정적 스코프)

🧠

상태 기억

클로저를 이용하면 함수가 끝난 뒤에도 내부 변수(c)를 계속 기억해서 상태를 유지할 수 있다.

🔐

데이터 은닉

JS에는 접근제어자(private)가 없다. 클로저로 외부에서 직접 접근 못하는 변수를 만들 수 있다.

09_closure.js 
// 렉시컬 스코프 — 선언된 위치 기준
const v = "global";
function f() { console.log(v); } // 선언시 v = "global"
function f2() {
  const v = "local";
  f(); // "global" 출력 (f는 global에서 선언됐으니까)
}
f2(); // "global" — f2 안의 v = "local"이지만 f는 무시!

// 클로저 — 상태를 기억하는 카운터
function f3() {
  let c = 0;
  return function () {
    return ++c; // f3이 끝나도 c는 살아있다!
  };
}

const counter = f3(); // 카운터 함수 생성
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3 — c가 계속 기억됨!

🎮 클로저 카운터 — 직접 실행해보기

카운터: 0
💡
버튼의 클릭 횟수 등 UI 상태 관리에 클로저가 활용된다. React의 useState도 내부적으로 클로저 원리를 사용한다.
10

🗂️ 객체 (Object)

관련 데이터와 기능을 묶은 단위. { key: value } 형태로 작성.

조작문법예시
읽기점 표기법 / 대괄호obj.name / obj["name"]
추가직접 할당obj.language = "JS"
수정직접 할당obj.numbers = [1,2,3]
삭제delete (거의 안 씀)delete obj.trash

값 목록Object.values()배열로 반환
키+값 쌍Object.entries()[[key,val], ...]
10_object.js 
const program = {
  name: "부자로 만들어주는 로또 생성기",
  numbers: [],
  pickNumber() {          // 메서드 (함수가 값인 프로퍼티)
    console.log("숫자를 뽑았습니다!");
  },
};

program.language = "JS";          // 프로퍼티 추가
program.numbers = [1,2,3,5,8];    // 프로퍼티 수정
delete program.trash;             // 프로퍼티 삭제

// Object 유틸리티 메서드
console.log(Object.keys(program));    // ["name","numbers","pickNumber","language"]
console.log(Object.values(program));  // 각 값들의 배열
console.log(Object.entries(program)); // [[key,value], ...] 쌍

// entries로 key + value 동시에 순회
for (const [k, v] of Object.entries(program)) {
  console.log(k, v);
}

📋 배열 (Array)

순서가 있는 데이터 목록. 스택(Stack)과 큐(Queue)처럼 활용할 수 있다.

메서드위치동작자료구조
push(v)맨 뒤추가🥞 Stack
(후입선출 LIFO)
pop()맨 뒤꺼내기
unshift(v)맨 앞추가🚌 Queue
(선입선출 FIFO)
shift()맨 앞꺼내기
splice(start, n)중간n개 잘라내기—
11_array.js 
const arr = [1, 2, 3, 4, 5];

// 스택 (Stack) — 후입선출 LIFO
arr.push(11);              // 맨 뒤에 추가 → [1,2,3,4,5,11]
const p = arr.pop();       // 맨 뒤 꺼내기 → p=11, arr=[1,2,3,4,5]

// 큐 (Queue) — 선입선출 FIFO
arr.unshift("a");           // 맨 앞에 추가 → ["a",1,2,3,4,5]
const s = arr.shift();      // 맨 앞 꺼내기 → s="a", arr=[1,2,3,4,5]

// splice(시작인덱스, 개수) — 중간 잘라내기
console.log(arr.splice(1, 2)); // [2, 3] 반환, arr=[1,4,5]

// 배열의 인덱스는 숫자 키
console.log(Object.keys(arr)); // ["0", "1", "2", ...]
ℹ️
JS 배열에서 push→pop은 스택(LIFO), push→shift는 큐(FIFO). 배열 메서드를 조합하면 대부분의 자료구조를 구현할 수 있다.

⚡ 배열 고차함수 (Higher-Order Functions)

함수를 인자로 받는 배열 메서드들. map / filter / reduce / find / sort — 반복문 없이 데이터를 가공하는 현대적인 방식.

map
변환 — 각 요소에 함수를 적용해 새 배열을 반환. 원본 불변.
↓
filter
걸러내기 — 조건이 true인 요소만 모아 새 배열을 반환.
↓
reduce
누산 — 배열을 순회하면서 값을 하나로 합산. 합계, 최댓값, 객체 생성 등 만능.
↓
find / findIndex
탐색 — 조건에 맞는 첫 번째 요소(또는 인덱스)를 반환. 없으면 undefined / -1.
↓
sort / some / every
정렬 / 판단 — sort는 원본 배열 정렬, some은 하나라도 true, every는 모두 true.
12_hof.js 
const nums = [1, 2, 3, 4, 5, 6];
const users = [
  { name: "윌리엄", age: 28, score: 92 },
  { name: "김자바",  age: 17, score: 45 },
  { name: "이스크",  age: 22, score: 78 },
  { name: "박노드",  age: 15, score: 60 },
];

// map — 모든 점수를 1.1배로 올린 새 배열
const boosted = users.map(u => ({ ...u, score: u.score * 1.1 }));

// filter — 성인(18세 이상)만 추출
const adults = users.filter(u => u.age >= 18);
// [{ name: "윌리엄" ... }, { name: "이스크" ... }]

// reduce — 평균 점수 계산
const avg = users.reduce((acc, u) => acc + u.score, 0) / users.length;
// acc: 누적값(초기값 0), u: 현재 요소

// find — 조건 맞는 첫 요소
const top = users.find(u => u.score >= 90); // { name: "윌리엄" ... }

// sort — 점수 내림차순 정렬 (원본 변경됨!)
const sorted = [...users].sort((a, b) => b.score - a.score);
// (b-a: 내림차순, a-b: 오름차순)

// some / every
const hasMajor  = users.some(u  => u.age >= 18);  // true (한 명 이상 성인)
const allPass   = users.every(u => u.score >= 40); // true (모두 40점 이상)

// 체이닝 — 성인 중 합격자(60점↑)의 이름 목록
const result = users
  .filter(u => u.age >= 18)
  .filter(u => u.score >= 60)
  .map(u => u.name);
// ["윌리엄", "이스크"]

이벤트 위임

부모에 이벤트를 달고 e.target으로 어떤 자식인지 구분. 동적으로 생성되는 요소에 효율적으로 이벤트를 붙이는 패턴.

14_event.js 
// 기본 이벤트 등록
const btn   = document.querySelector('#myBtn');
const input = document.querySelector('#myInput');

btn.addEventListener('click', (e) => {
  console.log('클릭됨!', e.target);  // e.target = 클릭된 요소
});

// input 이벤트 — 값이 바뀔 때마다 실행
input.addEventListener('input', (e) => {
  console.log(e.target.value); // 현재 입력값
});

// keydown — 엔터키 감지
input.addEventListener('keydown', (e) => {
  if (e.key === 'Enter') {
    console.log('엔터 입력!', e.target.value);
  }
});

// 이벤트 위임 — ul에만 달아서 li 클릭 감지
const list = document.querySelector('ul');
list.addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') {       // li 클릭인지 확인
    e.target.classList.toggle('done'); // done 클래스 토글
  }
});

// 이벤트 한 번만 실행 (once)
btn.addEventListener('click', () => {
  console.log('딱 한 번만 실행!');
}, { once: true });

// form submit 기본 동작(페이지 새로고침) 막기
document.querySelector('form').addEventListener('submit', (e) => {
  e.preventDefault(); // 새로고침 막음
  console.log('폼 제출 처리!');
});

⏳ 비동기 / Promise / async-await

서버에 데이터를 요청하거나 타이머를 걸 때처럼, 결과가 나중에 오는 작업을 처리하는 방법.

동기(Sync) 코드가 위에서 아래로 순서대로 실행됨. 한 줄이 끝나야 다음 줄 실행.
비동기(Async) 기다리지 않고 다음 코드를 먼저 실행. 결과가 오면 나중에 처리. (setTimeout, fetch, 파일 읽기 등)
.then / await 비동기 작업이 성공했을 때 실행할 코드. Promise가 resolve(이행) 상태가 됨.
.catch / try-catch 비동기 작업이 실패했을 때 실행할 코드. Promise가 reject(거부) 상태가 됨.
15_async.js 
// 비동기의 핵심 — 순서 확인
console.log("1️⃣ 시작");
setTimeout(() => console.log("3️⃣ 타이머 (1초 후)"), 1000);
console.log("2️⃣ 끝");
// 출력: 1️⃣ → 2️⃣ → (1초 후) 3️⃣

// ── Promise 기본 ──
const p = new Promise((resolve, reject) => {
  const ok = Math.random() > 0.5;
  if (ok) resolve("✅ 성공!");   // 성공 시 resolve 호출
  else   reject("❌ 실패!");    // 실패 시 reject 호출
});

p
  .then(result => console.log(result))  // 성공 처리
  .catch(err    => console.log(err))     // 실패 처리
  .finally(() => console.log("항상 실행")); // 성공/실패 상관없이

// ── async / await — Promise를 동기처럼 작성 ──
const fetchUser = (id) =>
  new Promise(resolve =>
    setTimeout(() => resolve({ id: id, name: "윌리엄" }), 500)
  );

async function loadUser() {
  try {
    console.log("로딩 중...");
    const user = await fetchUser(1); // Promise 결과를 기다림
    console.log("유저:", user);        // { id: 1, name: "윌리엄" }
  } catch (err) {
    console.log("에러:", err);
  }
}
loadUser();

// ── Promise.all — 여러 비동기를 동시에 ──
const [u1, u2, u3] = await Promise.all([
  fetchUser(1),
  fetchUser(2),
  fetchUser(3),
]); // 셋 다 완료될 때까지 기다림 (병렬 실행!)

💡
async/await는 Promise의 문법적 설탕(Syntactic Sugar)! 내부적으로는 똑같은 Promise를 사용하지만, 코드를 훨씬 읽기 쉽게 작성할 수 있다. 실무에서는 대부분 async/await를 선호한다.
